Reference

Welcome! Here you'll find everything you need to use NameToColor in your projects. All functions are exposed globally after including the script — no imports needed.

Optional Language Packs

The core NameToColor.js remains English. Data-only language files are loaded after it to add translated color names, input keywords, theme aliases, and localized return labels. The Brazilian Portuguese pack contains 139 common named-color entries, all 10 mood labels, and aliases for all 69 themes. Native English and every loaded language remain available at the same time. 

<script src="NameToColor.js"></script>
<script src="NameToColor.ptBR.js"></script>
generateColor("red");
generateColor("vermelho");
// → both return "#ff0000"

generateThemePalette("natureza");
// → native Nature palette

mood("#4682B4", "pt-BR");
// → ["Calmo", "Corporativo", "Sofisticado"]

colorName("#ff0000", "pt-BR");
// → "Vermelho"

Omitting the optional locale preserves the original English output. If a translated color name is unavailable, name lookup falls back to English. Use listNameToColorLanguages() to list loaded packs and registerNameToColorLanguage(pack) to add future data-only languages.

Color pair format

The English database and every translation pack store names in direct hexadecimal-to-array pairs. Keys contain exactly six hexadecimal digits and omit the leading #; public functions still return standard #rrggbb colors. 

const colorDatabase = {
    "000000": ["Black"],
    "00FFFF": ["Aqua", "Cyan", "Spanish Sky Blue"]
};

registerNameToColorLanguage({
    locale: "example",
    colorNames: {
        "ff0000": ["Translated Red"]
    }
});

🎯 generateColor(input)

generateColor(input) → String | String[]

The heart of the library. Accepts a wide variety of input types and returns a hexadecimal color string (#rrggbb or #rrggbbaa). When the input is an array, returns an array of color codes by recursing into each element. The result is deterministic for the same input (except for "random").

Input Types

Type Description Example
Array (recursive) Calls generateColor on each element, returns an array. Nested arrays also work. generateColor(["Lucas", 42]) → ["#…", "#…"]
Known color name Matches against an internal database of 2000+ named colors (case-insensitive, with contains and fuzzy matching) generateColor("Absolute Zero") → "#0048BA"
Numeric Index Looks up the hexadecimal key at that declaration-order position; if out of range, generates a deterministic color from the number generateColor(1) → "#0048BA"
Object Any non-array, non-element object is converted to string and processed generateColor({toString:()=>"Lucas"})
HEX Accepts #rgb, #rrggbb, with or without # generateColor("#ff00aa") → "#ff00aa"
RGB / RGBA Parses CSS-style rgb() and rgba() strings generateColor("rgba(255,99,71,0.5)") → "#ff634780"
CSS named color Any color name the browser recognizes (tomato, rebeccapurple, etc.) generateColor("tomato") → "#ff6347"
Random Returns a different random color each time generateColor("random") → "#a3f2c1"
Generic text Deterministic hash-based color for any arbitrary text generateColor("Lucas") → always the same color
Multiple words Splits into individual words. Words starting with dark, light, bright, or random are reordered as modifiers after their target color. Then each word is blended iteratively (75% current, 25% next). darker/lighter/brighter count as two modifiers; random generates a random HSL variation of the current color generateColor("Dark Zonaro") → darkened Zonaro
generateColor("Random Red") → random red variation
HTML Element Applies the generated color to the element's style.color using its textContent generateColor(document.getElementById("name"))
Empty / null / undefined Returns a random color for falsy inputs generateColor("") → "#a3f2c1" (random)

Processing Order

When you call generateColor(), the library evaluates the input in this order:

  1. HTML Element — extracts textContent and recurses
  2. Array — maps each element recursively
  3. Numeric Index — looks up the hexadecimal key at that declaration-order position; deterministic fallback if out of range
  4. Object — converts to string and recurses
  5. Empty input — returns a random color
  6. "random" — generates a random hex color
  7. CSS valid color — lets the browser resolve it
  8. HEX string — parses and normalizes
  9. RGB string — parses rgb(r, g, b)
  10. RGBA string — parses rgba(r, g, b, a)
  11. Multiple words — splits and reorders modifiers (dark/light/bright/random), then blends each word iteratively (75/25 ratio). random applies a random HSL variation instead of a fixed blend
  12. Exact name match — searches the internal database for an exact normalized match
  13. Contains name match — searches for partial name matches
  14. Levenshtein fuzzy match — finds the closest name within distance threshold
  15. Deterministic hash — fallback for any remaining text

🎨 generateReadableColor(input)

generateReadableColor(input) → [textColor, bgColor]

Returns a pair of colors — a background and a text color — where the text is always readable against the background, with a WCAG AA contrast ratio ≥ 4.5:1. And it doesn't just use pure black or white — it blends the base color to keep everything harmonious.

How It Works

  1. Generates the base background color with generateColor(input).
  2. Calculates the relative luminance of that background (WCAG formula).
  3. Chooses white or black as the starting text color — whichever has the highest contrast.
  4. Uses binary search to find the minimum blend ratio that achieves 4.5:1 contrast.
  5. Guarantees at least 50% blend — so the text color always retains some of the original hue.
💡 Pro tip: Use it directly on HTML elements for instant readable styling: 
const el = document.getElementById("username");
const [textColor, bgColor] = generateReadableColor(el.textContent);
el.style.color = textColor;
el.style.backgroundColor = bgColor;

// Or loop through multiple elements:
document.querySelectorAll(".tag").forEach(el => {
    const [text, bg] = generateReadableColor(el.textContent);
    el.style.color = text;
    el.style.backgroundColor = bg;
});

Examples

// With "tomato" — a warm, readable blend
generateReadableColor("tomato");
// → ["#b24338", "#ff6347"]  (blended text, not pure black)

// With "black" — still readable without pure white
generateReadableColor("black");
// → ["#4d4d4d", "#000000"]  (blended text, not pure white)

// With a light blue — contrast ≥ 4.5:1 guaranteed
generateReadableColor("#85d8fd");
// → ["#152228", "#85d8fd"]  (dark text, light background)
📌 Note: The returned array is always [textColor, backgroundColor]. Apply them to color and background-color respectively.

📚 listColors(pageNumber?, pageSize?)

listColors(pageNumber?, pageSize?) → { items, pageNumber, pageCount, totalItems }

Returns the entire internal hexadecimal-to-names database in a paginated format. Each color name appears as its own item (pairs with multiple names are flattened). Each item contains Color (name) and Hexadecimal, with the regular leading #.

Parameters

Parameter Type Default Description
pageNumber Number 1 The page to retrieve (1-indexed)
pageSize Number all items Number of items per page

Return Value

Property Type Description
items Array Array of { Color, Hexadecimal } objects
pageNumber Number Current page number
pageCount Number Total number of pages
totalItems Number Total items in the database

Examples

// Get all colors (no pagination)
listColors();
// → { items: [...], pageNumber: 1, pageCount: 1, totalItems: 2362 }

// Page 2 with 10 items per page
listColors(2, 10);
// → { items: [...], pageNumber: 2, pageCount: 237, totalItems: 2362 }

// Useful for building color pickers, tables, or browsers
const result = listColors(1, 20);
result.items.forEach((item, index) => {
    console.log(index, item.Color, item.Hexadecimal);
});

🎯 generateInvertedColor(input)

generateInvertedColor(input) → String

Returns the negative (inverted) color of the input. Each RGB channel is subtracted from 255, producing a photographic negative effect.

Examples

// Red → Cyan
generateInvertedColor("#ff0000");
// → "#00ffff"

// Black → White
generateInvertedColor("black");
// → "#ffffff"

// Gold → Deep blue
generateInvertedColor("gold");
// → "#0028ff"

🔄 generateComplementary(input)

generateComplementary(input) → String

Returns the complementary color — the exact opposite on the color wheel (180° away in HSL hue). Saturation and lightness are preserved.

Examples

// Red → Cyan
generateComplementary("#ff0000");
// → "#00ffff"

// Blue → Yellow
generateComplementary("#0000ff");
// → "#ffff00"

// Gray → Gray (no hue to complement)
generateComplementary("#808080");
// → "#808080"

🔺 generateTriadic(input)

generateTriadic(input) → [base, 120°, 240°]

Returns an array of 3 colors equally spaced on the color wheel (120° apart). The first color is the generated base color.

Examples

// Red → Red, Green, Blue (RGB!)
generateTriadic("#ff0000");
// → ["#ff0000", "#00ff00", "#0000ff"]

// Gold → Gold, Cyan, Purple
generateTriadic("gold");
// → ["#ffd700", "#00ffd9", "#d900ff"]

⬜ generateSquare(input)

generateSquare(input) → [base, 90°, 180°, 270°]

Returns an array of 4 colors equally spaced on the color wheel (90° apart) — a tetradic/square color scheme.

Examples

// Red → Red, Lime, Cyan, Violet
generateSquare("#ff0000");
// → ["#ff0000", "#80ff00", "#00ffff", "#7f00ff"]

✂️ generateSplitComplementary(input)

generateSplitComplementary(input) → [base, 150°, 210°]

Returns an array of 3 colors: the base color and two colors adjacent to its complement (150° and 210° from the original). This creates a softer contrast than a direct complementary scheme.

Examples

// Red → Red, Spring Green, Azure
generateSplitComplementary("#ff0000");
// → ["#ff0000", "#00ff80", "#007fff"]

🎨 generateMonochrome(input, count?)

generateMonochrome(input, count?) → [color, …]

Generates a monochrome palette with count colors (default 5, min 2, max 21). All colors share the same hue and saturation — only lightness varies, alternating lighter and darker than the original.

How ordering works

  • Colors are returned sorted from lightest to darkest (highest to lowest lightness).
  • The original color is included in its correct position within the sorted order.
  • For pure black or white, a grayscale gradient is generated.

Examples

// Red, 5 colors (default) — lightest to darkest
generateMonochrome("#ff0000");
// → ["#ffa8a8", "#ff5757", "#ff0000", "#a80000", "#570000"]
//    mais claro  ↑        original   ↓        mais escuro

// Gold, 3 colors
generateMonochrome("gold", 3);
// → ["#ffec80", "#ffd700", "#806c00"]
//    +claro     original   +escuro

// Black, 5 colors (grayscale)
generateMonochrome("black", 5);
// → ["#cccccc", "#999999", "#666666", "#333333", "#000000"]

generateThemePalette(theme, count?)

generateThemePalette(theme, count?) → String[]

Generates a deterministic semantic palette for a native English or loaded translated mood, place, material, audience, industry, season, or descriptive phrase. The function returns five lowercase #rrggbb colors by default; count is clamped from 2 to 21. Blank input returns [].

Resolution order

  1. Full canonical name or alias after case, accent, punctuation, whitespace, and camelCase normalization.
  2. The longest recognized multi-word alias inside the phrase.
  3. An exact single-word alias.
  4. Conservative typo matching: one edit for 5–7 character words and two edits for longer words.

Concrete subjects outrank moods when a phrase contains equally specific matches. Unknown non-empty text receives a deterministic analogous palette rather than an error. Known themes use five curated anchors and other sizes are interpolated in HSL using the shortest path around the hue wheel.

Examples

// Canonical theme
generateThemePalette("Nature");
// → ["#1b4332", "#2d6a4f", "#52b788", "#8d6e4f", "#f1faee"]

// Custom size
generateThemePalette("water", 7);
// → seven colors sampled from the curated Water anchors

// Aliases and typo tolerance resolve to the same palette
generateThemePalette("café");
generateThemePalette("cofee brand");
generateThemePalette("espresso");
// → Coffee palette

// Recognized alias inside a descriptive phrase
generateThemePalette("industrial website");
// → Industry palette

// Unknown themes are still deterministic
generateThemePalette("an entirely unknown subject");
// → analogous fallback palette
Direction matters: mood(color) classifies a color into one or more moods. generateThemePalette(theme) performs the inverse kind of task by generating colors from a semantic concept. The existing mood() rules remain unchanged.

🦾 isReadableForBlindness(colorA, colorB, type)

isReadableForBlindness(colorA, colorB, type) → Object

Checks whether two colors are readable (accessible) for various types of color blindness, simulating how the colors are perceived by a person with that condition and calculating the WCAG 2.1 contrast ratio between the simulated perceptions.

Parameters

Parameter Type Description
colorA * First color (any value accepted by generateColor())
colorB * Second color (any value accepted by generateColor())
type string Type of color blindness: "protanopia" (red-deficient), "deuteranopia" (green-deficient), "tritanopia" (blue-deficient), or "all" (tests all three types)

Return Value

For a specific type, returns an object with:

Property Type Description
readable boolean true if simulated contrast ≥ 4.5:1 (WCAG AA)
contrast number The exact simulated contrast ratio (e.g. 5.4)
simulatedColorA string HEX color of colorA as seen by the color blind person
simulatedColorB string HEX color of colorB as seen by the color blind person

For "all", returns an object mapping each of the three types with the structure above, plus an accessibleForAll flag that is true only if all three types pass.

Algorithm

  1. Linearization — Converts sRGB to linear RGB using the standard gamma expansion (threshold 0.04045)
  2. CVD Simulation — Multiplies linear channels by the simplified Brettel/Vienot matrices for the selected type
  3. Delinearization — Converts back to sRGB (threshold 0.0031308), multiplies by 255, and clamps
  4. WCAG Contrast — Calculates the relative luminance of each simulated color and the contrast ratio between them

Simulation Matrices

Type R' G' B'
Protanopia 0.56667·R + 0.43333·G 0.55833·R + 0.44167·G 0.24167·G + 0.75833·B
Deuteranopia 0.625·R + 0.375·G 0.70·R + 0.30·G 0.30·G + 0.70·B
Tritanopia 0.95·R + 0.05·G 0.43333·G + 0.56667·B 0.475·G + 0.525·B

Examples

// Red text on green background — illegible for protanopia
isReadableForBlindness("#ff0000", "#00ff00", "protanopia");
// → { readable: false, contrast: 1.2, simulatedColorA: "#c6c500", simulatedColorB: "#b0b187" }

// White on black — always readable, even for CVD
isReadableForBlindness("white", "black", "all");
// → { protanopia: {...}, deuteranopia: {...}, tritanopia: {...}, accessibleForAll: true }

// Real UI scenario with color names
isReadableForBlindness("tomato", "rebeccapurple", "deuteranopia");
// → { readable: true, contrast: 4.53, simulatedColorA: "#d6df51", simulatedColorB: "#575a85" }
💡 Tip: A contrast ratio below 3.0:1 is considered illegible for UI elements. A ratio below 4.5:1 fails WCAG AA for normal text. Use this function to verify your color pairs are accessible to people with color vision deficiencies.

🏷️ colorName(input, locale?)

colorName(input, locale?) → String | null

Returns the first color name whose hex exactly matches the generated color. Pass an optional loaded locale such as "pt-BR" to return its translated name. Omitting the locale preserves English; missing translations fall back to English.

Examples

// By name — returns the same name
colorName("Absolute Zero");
// → "Absolute Zero"

// By index — looks up the hex, finds the first name
colorName(1);
// → "Absolute Zero"

// By hex — exact database match
colorName("#0048BA");
// → "Absolute Zero"

// Explicit localized output
colorName("#ff0000", "pt-BR");
// → "Vermelho"

// Unknown color — returns null
colorName("unknown");
// → null

🏷️ colorNames(input, locale?)

colorNames(input, locale?) → String[]

Returns all color names (synonyms) for the hex that matches the input. Unlike colorName() which returns only the first name, this returns the full array of names for entries with multiple synonyms.

Examples

// Color with multiple names — returns all synonyms
colorNames("Aqua");
// → ["Aqua", "Cyan", "Spanish Sky Blue"]

// By hex — same result
colorNames("#00FFFF");
// → ["Aqua", "Cyan", "Spanish Sky Blue"]

// Localized synonyms
colorNames("#00FFFF", "pt-BR");
// → ["Ciano", "Água"]

// Unknown color — returns empty array
colorNames("unknown");
// → []

🎯 closestName(input, locale?)

closestName(input, locale?) → String | null

Like colorName(), but when there is no exact match in the database, it finds the nearest color by RGB Euclidean distance and returns its first name.

Examples

// Exact match — works like colorName()
closestName("#0048BA");
// → "Absolute Zero"

// Near match — finds closest by RGB distance
closestName("#ff6348");
// → "Tomato" (nearest color in the database)

// Any input — always returns the nearest color name
closestName("unknown");
// → nearest color name in the database

🎯 closestNames(input, locale?)

closestNames(input, locale?) → String[]

Like colorNames(), but with fuzzy fallback: if no exact match is found, finds the nearest color by RGB Euclidean distance and returns all its names.

Examples

// Exact match — works like colorNames()
closestNames("#00FFFF");
// → ["Aqua", "Cyan", "Spanish Sky Blue"]

// Near match — finds closest and returns all its names
closestNames("#ff6348");
// → ["Tomato"]

// Any input — always returns names of the nearest color
closestNames("unknown");
// → names of the nearest color

Luminance & Light/Dark helpers

const hex = generateColor("gold");
// hex → "#ffd700"

relativeLuminance(hex);
// → 0.699 (bright color)

isLight(hex);
// → true

isDark(hex);
// → false
📌 Note: isLight and isDark are complementary — a color is either light or dark, never both, never neither. The threshold is a relative luminance of 0.5.

Temperature (Warm/Cool) helpers

temperature(input) — 7 Temperature Levels
temperature(input) → String

Returns the temperature level of a color as one of 7 strings: VeryHot, Hot, Neutral Hot, Neutral, Neutral Cold, Cold, or VeryCold.

Uses a continuous two-factor scoring system that averages: HSL Hue (piecewise linear function: 0°–90° warm +1→0, 90°–210° cooling 0→-1, 210°–330° warming -1→0, 330°–360° warm 0→+1; achromatic colors get 0) and Direct RGB ((R - B) / 255, ranging from -1 to +1). The combined score ranges from -1 (coldest) to +1 (warmest) and is mapped to 7 distinct levels. 

// 🔥 VeryHot — hue 0° (+1), R >> B (+1)
temperature("red");
// → "VeryHot"

temperature("gold");
// → "VeryHot"

temperature("tomato");
// → "VeryHot"

// 🔥 Hot — warm hue, moderately warm
temperature("pink");
// → "Hot"

// 👍 Neutral Hot — slightly warm-leaning (boundary hue, R > B)
temperature("chartreuse");
// → "Neutral Hot"

// ⚪ Neutral — achromatic + balanced RGB
temperature("gray");
// → "Neutral"

// 👎 Neutral Cold — slightly cool-leaning
temperature("green");
// → "Neutral Cold"

temperature("magenta");
// → "Neutral Cold"

// ❄️ Cold — cool hue, moderately cool
temperature("aquamarine");
// → "Cold"

// ❄️ VeryCold — cool hue + R << B
temperature("blue");
// → "VeryCold"

temperature("cyan");
// → "VeryCold"
isHot(input) / isCold(input) — Boolean helpers

Both functions now delegate to temperature() internally.

// temperature("red") → "VeryHot"
isHot("red");
// → true

isCold("red");
// → false

// temperature("blue") → "VeryCold"
isCold("blue");
// → true

isHot("gray");
// → false (temperature("gray") → "Neutral")

isCold("gray");
// → false
🔥❄️ How it works: isHot and isCold are convenience wrappers around temperature():
  • isHot(input) — returns true when temperature(input) is "VeryHot", "Hot", or "Neutral Hot"
  • isCold(input) — returns true when temperature(input) is "VeryCold", "Cold", or "Neutral Cold"
  • When temperature() returns "Neutral", both isHot() and isCold() return false
See the temperature() section above for the full scoring algorithm details.

🎭 mood(input, locale?)

mood(input, locale?) → String[]

Returns an array of mood (atmosphere) names that best describe the generated color, based on mathematical rules in HSL space. Pass an optional loaded locale such as "pt-BR" to translate returned labels. Omitting it preserves English. If no mood matches, returns an empty array [].

How It Works

  1. Generates the color via generateColor(input) and converts it to HSL.
  2. Checks exact rules — if the color's H, S, L values fit one or more mood definitions exactly, those moods are returned.
  3. If no exact match is found, uses a proximity scoring system that calculates how close the HSL values are to each mood's ideal ranges, then returns moods above a threshold.
  4. If no mood scores high enough, returns [].

The 10 Moods

Mood HSL Rule (Exact) Description
Vibrant S ≥ 80, L 50–70, H 0–60 or 330–360 High saturation, medium lightness, warm hues
Futuristic S ≥ 90, L 45–55, H 180–220 or 280–320 Very high saturation, cool cyans/blues/purples
Fun S 65–90, L 65–80 High saturation, high lightness (bright & playful)
Soft Pastel S 15–40, L 80–95 Low saturation, high lightness (delicate & muted)
Calm H 160–260, S 20–50, L 60–85 Cool blue-green hues, serene atmosphere
Vintage H 20–50, S 30–50, L 40–60 Warm muted hues, medium saturation, retro feel
Organic H 30–45 or 90–140, S 40–60, L 40–55 Earthy greens & warm hues, natural feel
Sophisticated S 40–75, L 10–35 Medium saturation, deep elegant colors
Dark S 5–25, L 10–25 Near-black muted, very low saturation
Corporate (H 210–230 & S 10–30) or S < 10 (gray) Low-saturation blue or pure grays

Examples

// OrangeRed — vibrant warm color
mood("#FF4500");
// → ["Vibrant"]

// Pure blue — futuristic, intense
mood("#0000FF");
// → ["Futuristic"]

// Steel Blue — calm, corporate, sophisticated
mood("#4682B4");
// → ["Calm", "Corporate", "Sophisticated"]

// The same classification with Brazilian Portuguese labels
mood("#4682B4", "pt-BR");
// → ["Calmo", "Corporativo", "Sofisticado"]

// Near-black gray — corporate (pure gray)
mood("#1A1A1A");
// → ["Corporate"]

// White — corporate (S < 10)
mood("#FFFFFF");
// → ["Corporate"]

// Invalid input — returns empty array
mood("invalid color");
// → []
🎭 Proximity scoring: When a color doesn't fit any exact rule, the function calculates a fit score for each mood using multiplicative Gaussian decay across the constrained HSL dimensions. Only moods with a product score ≥ 0.3 (out of 1.0) are returned. This ensures that if any dimension is too far from the mood's range, the score drops sharply — keeping results meaningful and not overly broad.
